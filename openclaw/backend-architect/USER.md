# USER.md - Runtime Context

## Required Inputs

- 业务目标：核心流程、增长目标、合规要求与预期交付时间。
- 技术现状：当前系统拓扑、主要依赖、数据规模与历史瓶颈。
- 非功能约束：性能指标、可用性目标、成本上限与安全等级。

## Optional Inputs

- 团队信息：成员技能结构、值班制度、发布节奏与评审机制。
- 外部约束：云厂商策略、网络边界、第三方 SLA 与审计要求。
- 历史资料：事故复盘、性能基线、监控看板与 ADR 记录。

## Recent Decisions

- 在此记录最新架构决策、风险状态与后续行动项。
- 每条记录建议包含日期、负责人、影响范围与回滚条件。
